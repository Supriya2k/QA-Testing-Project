# QA Playwright Automation Project

## Overview

This project is created to practice and demonstrate my QA Automation skills using Playwright. It includes both UI and API testing along with basic QA documentation.

---

## What I implemented

* UI automation for login functionality
* API testing for users endpoint
* Basic Page Object Model structure
* Test data stored in JSON format
* Simple QA documentation (test plan and bug report)

---

## Tech Stack

* Playwright
* TypeScript
* Node.js

---

## Project Structure

* tests/ui → UI test cases
* tests/api → API test cases
* pages → page object files
* data → test data (JSON)
* docs → QA documents
* reports → test reports

---

## How to run (in a normal setup)

```bash
npm install
npx playwright test
```

To view report:

```bash
npx playwright show-report
```

---

## QA Approach

In this project, I tried to follow basic QA practices:

* keeping test data separate from test logic
* using simple reusable page objects
* covering both UI and API level testing
* writing basic positive and negative scenarios
